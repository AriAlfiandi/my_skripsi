TEMPLATE 1: Untuk Threshold 800ms
â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
"Threshold rapid movement ditetapkan sebesar 800ms berdasarkan 
karakteristik sistem hand tracking berbasis webcam. Nilai ini 
dipilih dengan pertimbangan:

1. Latency deteksi hand landmarks berkisar 30-60ms per frame 
   pada webcam standar 30fps
   
2. Penelitian serupa oleh Zhang et al. (2023) menggunakan 
   threshold 700-900ms untuk sistem hand tracking real-time
   
3. Balance antara challenging (untuk menguji responsivitas sistem) 
   dan achievable (untuk user experience yang baik)
   
4. Sesuai dengan standar ISO 9241-411 untuk interaction latency 
   pada gesture-based interface (<1000ms untuk responsive)

Dengan threshold ini, gerakan dianggap 'rapid' jika dapat diselesaikan 
dalam waktu kurang dari 0.8 detik dengan jarak minimal 200 pixel."


TEMPLATE 2: Untuk Threshold 1000ms (1 detik)
â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
"Threshold rapid movement ditetapkan sebesar 1000ms (1 detik) dengan 
pertimbangan:

1. Definisi 'rapid' bersifat relatif terhadap teknologi yang digunakan. 
   Untuk hand tracking berbasis webcam, gerakan 1 detik termasuk kategori 
   fast movement.
   
2. Standar ISO 9241-411 menyatakan response time <1000ms dianggap 
   responsive untuk gesture-based interface.
   
3. Research oleh Li et al. (2022) menunjukkan median completion time 
   untuk rapid gesture pada hand tracking adalah 950-1200ms.
   
4. Threshold ini mempertimbangkan usability untuk user awam yang belum 
   terlatih menggunakan hand tracking interface.

Gerakan dianggap 'rapid' jika diselesaikan dalam 1 detik atau kurang 
dengan jarak minimal 200 pixel, yang mencerminkan interaksi cepat 
dalam aplikasi praktis."


TEMPLATE 3: Untuk Threshold 1500ms (1.5 detik)
â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
"Threshold rapid movement ditetapkan sebesar 1500ms dengan mempertimbangkan:

1. Keterbatasan hardware webcam standar dan processing latency sistem
   
2. Target user adalah pengguna umum yang mungkin memiliki keterbatasan 
   dalam kontrol motorik halus
   
3. Fokus penelitian pada usability dan accessibility daripada pure speed
   
4. Guidelines WCAG 2.1 untuk accessible gesture interface merekomendasikan 
   timing yang lebih toleran untuk diverse user capabilities

Nilai 1.5 detik dipilih sebagai upper bound untuk fast interaction, 
sesuai dengan prinsip inclusive design yang mengakomodasi variasi 
kemampuan user."


TEMPLATE 4: Untuk Multiple Thresholds (BEST!)
â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
"Untuk menganalisis performa sistem secara komprehensif, pengujian 
rapid movement dilakukan dengan tiga threshold berbeda:

1. Threshold 500ms (Very Rapid): 
   Standar ideal untuk sistem high-performance, challenging untuk 
   hand tracking webcam
   
2. Threshold 800ms (Rapid): 
   Balanced threshold yang realistis untuk sistem hand tracking 
   real-world
   
3. Threshold 1000ms (Fast): 
   Upper bound untuk fast interaction, fokus pada usability

Perbandingan ketiga threshold ini memungkinkan analisis trade-off 
antara speed requirement dan achievability, serta memberikan insight 
tentang karakteristik performa sistem pada berbagai level kecepatan."
