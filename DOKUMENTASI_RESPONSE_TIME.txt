üìà METRIK YANG DITRACK
================================================================================

1. RESPONSE TIME PER GESTURE
   - Waktu dari gesture terdeteksi sampai action dieksekusi
   - Diukur dalam milidetik (ms)
   - Tersimpan untuk setiap deteksi
   - Dihitung: Average, Min, Max

2. FRAME PROCESSING TIME
   - Waktu yang dibutuhkan untuk memproses 1 frame
   - Dari start frame sampai end frame
   - Digunakan untuk menghitung FPS
   - Formula: FPS = 1000 / frame_processing_time_ms

3. STATISTIK YANG DIHITUNG:
   Per Gesture:
   - Total Detected
   - True Positive (TP)
   - False Positive (FP)
   - False Negative (FN)
   - Akurasi (%)
   - Response Time: Avg, Min, Max (ms)
   
   Keseluruhan:
   - Akurasi Sistem (%)
   - Response Time Rata-rata (ms)
   - Frame Processing Time Rata-rata (ms)
   - FPS

üîç INTERPRETASI RESPONSE TIME
================================================================================

KATEGORI RESPONSE TIME:
-----------------------
< 50ms   : Excellent (Sangat Responsif) ‚≠ê‚≠ê‚≠ê‚≠ê‚≠ê
50-75ms  : Good (Responsif) ‚≠ê‚≠ê‚≠ê‚≠ê
75-100ms : Fair (Cukup) ‚≠ê‚≠ê‚≠ê
100-150ms: Poor (Lambat) ‚≠ê‚≠ê
> 150ms  : Very Poor (Sangat Lambat) ‚≠ê

ANALISIS:
---------

Jika Response Time TINGGI (>100ms):
‚ùå Masalah:
   - Hardware kurang powerful
   - Resolusi kamera terlalu tinggi
   - Algoritma deteksi belum optimal
   - Background terlalu kompleks

‚úÖ Solusi:
   - Turunkan resolusi kamera (640x480)
   - Kurangi detectionCon (0.5 ‚Üí 0.3)
   - Gunakan background polos
   - Upgrade hardware (CPU/RAM)

Jika Response Time BAIK (<50ms):
‚úÖ Sistem sudah optimal!
   - Hardware mencukupi
   - Algoritma efisien
   - Kondisi ideal

PERBANDINGAN RESPONSE TIME ANTAR GESTURE:
------------------------------------------

Normal Range:
- Move Cursor: 30-40ms (tercepat, tracking kontinyu)
- Scroll: 35-50ms (deteksi jarak)
- Klik: 40-60ms (deteksi jarak presisi tinggi)
- Toggle: 45-65ms (deteksi kepalan, lebih kompleks)

Jika ada gesture jauh lebih lambat ‚Üí butuh optimasi

================================================================================
‚ùì TROUBLESHOOTING
================================================================================

Q: Response time selalu 0ms atau sangat rendah?
A: Bug! Pastikan frame_start_time diset sebelum processing.
   Cek kode di line: frame_start_time = time.time()

Q: Response time sangat tinggi (>200ms)?
A: Hardware lemah atau terlalu banyak proses berjalan.
   - Tutup aplikasi lain
   - Turunkan resolusi kamera
   - Cek CPU usage

Q: Response time tidak konsisten (30ms lalu 150ms)?
A: Normal untuk beberapa frame pertama (warm-up).
   - Jalankan 1-2 menit dulu sebelum test formal
   - Ambil data setelah sistem stabil

Q: FPS terlalu rendah (<15)?
A: - Turunkan resolusi kamera
   - Kurangi detectionCon
   - Upgrade hardware

Q: Response time berbeda setiap session?
A: - Pastikan kondisi sama (cahaya, jarak, hardware state)
   - Warming up setiap session
   - Gunakan sample size besar (>50)
